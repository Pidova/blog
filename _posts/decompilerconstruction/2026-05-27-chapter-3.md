## Pipeline

Lifting is the process of translating machine instructions into IL instructions while preserving semantics.

A typical lifting process follows these steps:
1. Dissassemble the instruction
2. Map architecture-specific registers to virtual registers
3. Identify implicit side effects (flags, memory, registers)
4. Emit one or more IL instructions representing the behavior

## DSL

When lifting assembly to IL, it is often helpful to create a Data Structure Language(DSL) to simplify the process.  
Use operator overloading and helper classes to lift complex instructions into smaller transformations.

**Reference the target ISA developer manual to understand instruction behavior and determine what needs to be implemented.**

### Branching

Branching is tricky, as if done incorrectly it will cause many side effects.  
Because each instruction in the target architecture is not guaranteed to map to exactly one IL instruction, 
we map the start of each IL instruction with its original address.

For indirect calls, we can reference all logged indirect targets as discussed in  
[Chapter 1]({% post_url decompilerconstruction/2026-05-27-chapter-1 %}).

## Example Walkthrough

We will lift with simple steps:
```
cmp eax, 0
je target
```

Step 1: Register mapping
```eax -> r1```

Step 2: Emit comparison
```
CMP r1, 0
```

Step 3: Emit branch condition
```
JUMPIF_EQUAL target
```

## CPU Specific
Example **syscall** you can do:

```
SCALL syscall, r1, 3
```
Scall stands for special call which is calling syscall. With it having 3 arguments (r1, r2, r3).
These can be used to flag or model architecture-specific effects associated with certain instructions.
